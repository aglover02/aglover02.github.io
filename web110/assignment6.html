Session 6 Networking Activity 1: Complete the following tutorials: This activity focused on understanding two reference models in computer networking: the OSI model and the TCP/IP model, as well as concepts related to Transmission Control Protocol (TCP), Internet Protocol (IP), and Domain Name System (DNS). The OSI model is a theoretical framework comprising seven layers, each with their own functions to allow for communication between various different devices over a network. But, the TCP/IP model, which is practical and widely used, combines the physical and data link layers into one and consists of four layers: Application, Transport, Internet, and Host-to-Network. TCP, an important protocol in the TCP/IP suite, ensures reliable, connection-oriented data transmission, offering features like stream data transfer, efficient flow control, and full-duplex operation. Meanwhile, IP, the backbone of internet communication, provides connectionless and unreliable packet delivery, requiring pairing with TCP for reliability. The DNS system, introduced to address the limitations of downloading large host files, resolves domain names to IP addresses, using a hierarchical naming scheme and distributed database of IP addresses and names. It involves processes where local DNS servers query root DNS servers, which delegate authority to other DNS servers until the IP address for a domain name is obtained. Through this activity, I gained a comprehensive understanding of networking models, protocols like TCP and IP, and the role of DNS in translating domain names to IP addresses. In my current or future environment, this knowledge will be useful for understanding and troubleshooting network communications, ensuring efficient data transmission, and addressing issues related to domain resolution. Whether working in IT support, network administration, or software development, a solid grasp of these networking fundamentals will enable me to effectively manage and improve network infrastructure and services.
